Background

Using devices such as Jawbone Up, Nike FuelBand, and Fitbit it is now possible to collect a large amount of data about personal activity relatively inexpensively. These type of devices are part of the quantified self movement – a group of enthusiasts who take measurements about themselves regularly to improve their health, to find patterns in their behavior, or because they are tech geeks. One thing that people regularly do is quantify how much of a particular activity they do, but they rarely quantify how well they do it. In this project, your goal will be to use data from accelerometers on the belt, forearm, arm, and dumbell of 6 participants. They were asked to perform barbell lifts correctly and incorrectly in 5 different ways. More information is available from the website here: http://groupware.les.inf.puc-rio.br/har (see the section on the Weight Lifting Exercise Dataset).

train_data<-train_data[,-c(1:6)]

Splitting in Training and Testing

I split the updated training dataset into a training dataset (70% of the observations) and a validation dataset (30% of the observations). I will use the latest to perform cross validation when creating the model.

inTrain = createDataPartition(y = train_data$classe, p = 0.7, list = FALSE)
training = train_data[inTrain, ]
validation = train_data[-inTrain, ]

Now, after the aplit it is important to check if the remiaing variables are highly correlated and if so, remove them or replace them with something like PCA.

correlations <- cor(training[, -53])
corrplot(correlations, order = "FPC", method = "color", type = "lower", tl.cex = 0.8, 
    tl.col = rgb(0, 0, 0))

The image above shows in darker colors the pair of predictors that are highly correlated to each other(excluding the main diagonal). It looks like we might have many varaibles that can be grouped together in a weigthed combination.

Hence my next step is to implement Principal Component Analysis to create new variables that summarize the ones identified above. Here careful is need since the new variables might be difficult to explain.

PCA

Using the caret package we preprocess the data leaving out the variable we want to predict (column 54) Then the predict function is used to apply the same PCA to the orignal data.

preProc <- preProcess(small_train[, -54], method = “pca”, thresh = 0.99) trainPC <- predict(preProc, small_train[, -54]) valid_testPC <- predict(preProc, small_valid[, -54])

Fiting a Random Forest model

I choose to apply a Random Forest model and the Cross validation method in the trainControl parameter.

modelFit <- train(small_train$classe ~ ., method = “rf”, data = trainPC, trControl = trainControl(method = “cv”, number = 4), importance = TRUE)

Below is the graph for the importance of each principal componet, PC1, PC18 and PC15 are shown to be the most important ones.

varImpPlot(modelFit$finalModel, sort = TRUE, type = 1, pch = 19, col = 1, cex = 1, main = " Individual Principal Components“)
